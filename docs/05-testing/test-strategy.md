# 测试策略

## 测试层级

- 单元测试：数据转换、聚合规则、版本管理、Token 统计。
- 集成测试：热点 API、OpenAI 兼容接口、数据库和系统凭据库。
- UI 测试：核心创作流程、复制、删除、备份与恢复。
- 打包测试：Windows 和 macOS 安装、升级、卸载和数据保留。

## 重点异常场景

- 公共热点 API 不可用或返回异常数据。
- 单个平台生成失败或超时。
- 用户取消生成。
- AI 返回非预期结构。
- 磁盘空间不足、图片丢失、数据库损坏。
- 备份文件损坏或版本不兼容。
- 用户选择的数据目录不存在、磁盘未连接或无写入权限。
- 数据目录异常时不得静默创建空数据库。
