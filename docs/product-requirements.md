# 自媒体工作台产品需求

> 状态：需求讨论中  
> 最近更新：2026-07-12

## 1. 产品定位

- 面向个人用户的本地自媒体创作工作台。
- 客户端形态为跨平台桌面应用，支持 Windows 和 macOS。
- 不包含注册、登录、多用户和数据隔离系统。
- 用户选择或输入主题、创作选项、参考素材和目标平台后，由 AI 生成各平台内容。

## 2. 首版目标平台

- 微信公众号
- 小红书
- 抖音
- 今日头条

用户必须主动勾选目标平台，系统不会默认生成全部平台内容。

## 3. 核心流程

1. 手动获取热点。
2. 将内容录入素材库，可跳过。
3. 选择热点或输入自定义主题进行创作。
4. 在内容分发页查看、复制并手动记录分发状态。

## 3.1 首次启动

1. 用户选择并验证业务数据目录，此步骤必须完成。
2. 用户可配置 OpenAI 兼容接口并测试连接，也可以跳过。
3. 未配置 AI 时仍可查看原始平台热点和使用素材库；AI 聚合及内容创作入口提示先配置模型。

## 4. 模块需求

### 4.1 热点

- 热点数据直接通过 NewsNow API 获取，不依赖或集成 TrendRadar。
- TrendRadar 仅作为请求重试、数据校验和历史存储逻辑的参考。
- 首版只内置一个 NewsNow 热点源，不提供热点源新增、编辑、启停和删除页面。
- 数据管理高级设置允许修改这一 NewsNow API 地址，默认使用公共实例。
- 技术架构预留统一热点源适配器接口，后续版本支持接入不同协议的热点服务。
- 初步数据源包括今日头条、百度、微博、抖音、知乎。
- 只由用户手动刷新，不进行定时自动抓取。
- 保存标题、排名、热度、链接、来源和抓取时间等信息。
- 只有进入创作流程后，用户才可选是否读取热点链接正文；默认关闭，需手动勾选。
- 首版通过通用 HTTP HTML 获取和正文提取读取链接，不使用浏览器自动化。
- 不处理登录、验证码和强反爬页面；失败时退回热点标题与摘要，并允许用户手动粘贴正文。
- 成功读取的正文只保存到当前项目热点快照，项目内不重复请求；首版不建立全局正文缓存。
- 聚合热点包含多个来源时，由用户勾选需要读取正文的来源，默认不选择。
- 提供“选择排名最高的 3 个来源”快捷操作。
- AI 将同一事件的跨平台数据聚合为一个热点主题，同时保留各平台来源、排名和热度变化。
- 刷新时先抓取并保存原始平台榜单，再调用 AI 进行聚合。
- 原始榜单抓取成功后，即使 AI 聚合失败也可正常查看，并允许用户手动重新聚合。
- NewsNow 平台请求最大并发数为 3，单请求超时 10 秒，失败最多重试 2 次。
- 响应使用 Zod 校验，链接要求 HTTPS 并通过平台域名白名单校验。
- 部分平台失败时保存其他平台的成功数据，并展示各平台刷新状态。
- 聚合热点和平台榜单共享统一时间筛选，默认选择“今天”。
- 不设置独立历史热点页签，历史数据通过时间、关键词和平台筛选直接查询。
- 平台榜单按平台分别展示，不混排不同平台排名。
- 热点数据永久保留，用户可手动清除。
- 每隔 30 天提醒用户清理热点；不自动删除，可选择立即清理或 30 天后提醒。
- 清理范围支持 30 天前、90 天前、自定义日期前和全部，默认选择 30 天前。
- 清理前展示预计删除数量，并要求二次确认。

### 4.2 素材库

- 定位为创作参考文收藏夹。
- 首版只支持手动粘贴文本，不支持链接解析或本地文件上传。
- 一次创作可选择多条素材。
- 素材可标记为重点参考或普通参考。
- 当参考内容超过模型上下文限制时，优先保留重点素材，普通素材可自动截取或压缩摘要。
- 发生压缩时需要提示用户，并允许查看实际送入模型的参考内容。
- 创作项目选择素材时，将标题、正文、标签和参考优先级保存为项目专属快照。
- 后续修改或删除素材库原文，不影响已有项目及历史版本。
- 用户重新选择素材时才以素材库最新内容更新项目快照。
- 素材标题和正文必填，标签可选。
- 标签使用自由文本，仅用于内容识别和创作参考，不建立可复用标签库。
- 素材列表摘要直接截取正文前部，不额外调用 AI 生成摘要。
- 删除素材时二次确认，然后直接删除，不进入回收站。
- 支持批量选择并删除素材，批量删除同样需要二次确认。
- 素材库页面不提供“用于创作”入口。
- 用户只在创作设置中搜索并勾选参考素材；生成母稿时，素材快照才作为 AI 上下文。

### 4.3 创作

#### 创作输入

- 支持从热点列表选择主题。
- 支持直接输入自定义主题。
- 创作项目可以不关联热点。首版不提供 AI 联网搜索；用户通过热点、素材库或手动输入补充资料，联网搜索放入后续版本。
- 一个创作项目可以关联多个热点，在生成母稿时共同作为参考上下文。
- 热点进入创作项目时，保存聚合标题、AI 摘要、来源平台、原始标题、链接、排名和当时热度的项目快照。
- 热点历史后续更新或清理，不影响已有创作项目中的主题来源。
- 支持输入用户自己的见解。
- 支持勾选素材库内容作为参考。
- 热点和素材均通过可搜索的选择弹窗勾选，创作设置页只展示已选摘要，避免列表过长。
- 支持勾选一个或多个目标平台。
- 提供目标受众、内容风格、篇幅、表达立场、营销目的等可选项。
- 创作选项对本次所选的全部平台统一生效，不做逐平台配置。

#### 生成流程

- 同一主题先生成一套母稿，再基于母稿生成用户勾选的平台版本。
- 新建项目按“创作设置 → 生成并确认母稿 → 生成平台内容 → 上传图片”推进。
- 项目详情页签始终可查看，但未满足前置条件时禁用对应生成操作。
- 母稿在创作页展示并允许编辑。
- 修改母稿后，已生成的平台内容保持不变，由用户手动刷新需要更新的内容。
- 首次生成时一次性生成平台的全部字段。
- 每个内容模块提供刷新按钮，可单独重新生成。
- 各平台并行且独立生成；单个平台失败不影响其他结果，并可单独重试。
- 生成过程允许取消；取消后只保存已经完整生成的平台结果，不保存未完成内容。
- AI 输出超过平台字符限制时自动压缩，不要求用户选择处理方式。
- 自动压缩后仍超限，正文按完整句子边界裁剪，标题等短字段按字符上限裁剪。
- 写入编辑器的最终生成结果必须满足 PlatformSpec 限制。

#### 内容编辑格式

- 微信公众号和今日头条正文使用富文本编辑器，支持标题层级、段落、加粗、列表、引用和图片占位。
- 小红书和抖音内容使用纯文本编辑体验。
- 富文本内部保存结构化 JSON，并同步维护纯文本副本。
- 分发复制微信公众号和今日头条正文时，同时写入 HTML 和纯文本剪贴板格式，目标平台支持时保留排版。

#### 项目与版本

- 每次创作保存为一个主题项目。
- 项目包含母稿、平台内容、图片提示词、用户上传图片、当前版本和上一版本。
- 版本以整个创作项目为单位，只保留当前版本和上一版本。
- 自动保存只保存当前编辑稿、防止内容丢失，不创建新版本。
- 用户主动点击“保存版本”时，原当前版本整体移为上一版本，当前编辑稿成为新的当前版本。
- AI 刷新、手动编辑和图片上传本身不创建版本。
- 上一版本以只读方式查看。
- 用户可将上一版本恢复为当前编辑稿；恢复动作不立即建立新版本，仍需用户主动点击“保存版本”。
- 项目状态由系统自动计算，用户不可手动设置。
- 状态包括草稿、创作中、待分发、分发中和已完成。
- 生成失败通过独立错误标记展示，不改变项目主状态定义。
- 首版支持复制创作项目。
- 复制当前编辑稿、素材快照、热点快照、平台内容和图片资产引用。
- 不复制历史版本、分发记录和已发布状态。
- 副本替换图片只修改引用，不影响原项目。
- 创作设置、母稿、平台内容和图片信息采用自动保存编辑稿。
- 用户停止输入约 1 秒后写入本地数据库。
- 页面顶部显示保存中、已保存或保存失败；存在保存失败时，离开页面需要二次确认。
- 删除创作项目时二次确认，然后直接删除，不进入回收站。
- 删除创作项目时，同时删除其母稿、平台内容、版本、图片引用和分发记录。
- 项目图片资产没有其他引用时，同时物理删除本地文件。
- 已产生的 Token 日统计和保留期内日志不因项目删除而回退。

#### 图片

- 首版不调用图片生成模型，只生成封面和配图提示词。
- 用户可在生成后上传对应成品图，用于项目留存和后续查看。
- 每种平台尺寸由用户分别上传成品图，应用不负责自动裁剪。
- 图片必须归属于某个创作项目，并进一步关联目标平台和用途。
- 每个平台的每种封面规格只保留一张当前图片。
- 内容配图允许上传多张，并支持排序、替换和删除。
- 每张图片保存所属项目、平台、用途、比例、提示词和本地文件信息；内容配图可保存顺序和备注。
- 内容配图默认按顺序管理，并可选绑定到正文某一段之后。
- 分发页根据绑定位置在正文中展示图片占位提示。
- 图片文件作为独立资产保存，编辑稿、当前版本和上一版本通过引用关联图片。
- 删除或替换图片时只移除当前引用；仍被任一保留版本引用的文件不得物理删除。
- 图片不再被任何编辑稿或版本引用后，由本地清理逻辑自动删除文件。
- 内容配图提示词数量由系统根据正文篇幅和段落结构自动决定，每个平台最多 8 个。
- 每个提示词默认绑定对应正文段落。
- 用户可单独刷新、删除或新增配图提示词，并上传多张成品图。

#### 平台字段

| 平台 | 首版生成字段 |
|---|---|
| 微信公众号 | 标题、摘要、正文、标签、21:9 封面提示词、1:1 分享图提示词、正文配图提示词 |
| 小红书 | 标题、正文、话题标签、16:9 或 9:16 封面提示词、正文配图提示词 |
| 抖音 | 选题、标题、作品描述、话题、口播稿、分镜、9:16 封面提示词 |
| 今日头条 | 标题、摘要、正文、标签、16:9 封面提示词、正文配图提示词 |

### 4.4 内容分发

- 首版不做自动发布，避免平台风控；后续版本再评估。
- 分发页只展示创作页的最终内容，并提供便捷复制。
- 各平台字段分别提供复制，不提供“一键复制全部文本”。
- 图片提供单独复制和打开所在位置。
- 所有内容编辑和制作均在创作页完成，分发页不可编辑正文。
- 分发页读取创作项目最新的自动保存编辑稿，而不是最近一次主动保存的版本。
- 编辑稿尚未保存为版本时，分发页显示提示但仍允许复制。
- 分发状态由用户手动标记。
- 分发页保存分发记录。
- 分发状态包括待分发、已复制和已发布。
- 用户复制任一平台字段后，该平台分发任务自动变为已复制。
- 已发布必须由用户手动标记，可选填写发布时间和作品链接。
- 已发布内容后续发生修改时，保留已发布状态并显示“发布后有修改”。

### 4.5 统计

- 统计热点数量、素材数量和创作主题数量。
- Token 用量只按日期统计。
- Token 指标包括输入 Token、输出 Token和总 Token。
- 首版不估算模型费用。
- 统计功能使用独立页面，不设置工作台概览页。

### 4.6 设置

- AI 只支持 OpenAI 兼容接口。
- 首版只保存一套接口配置和一个默认模型，不建立多供应商配置列表。
- 不为特定供应商硬编码接口逻辑；小米 MiMo 作为首个兼容验证目标。
- 用户手动填写 Base URL、API Key 和模型名。
- AI 高级设置提供可选的上下文窗口 Token 和最大输出 Token。
- 未填写时默认使用上下文 32,000、最大输出 8,000。
- 提供测试连接功能。
- 测试连接发送一次最小模型生成请求，不依赖模型列表接口。
- 测试结果显示响应、耗时和 Token；测试产生的 Token 计入当天统计。
- API Key 加密保存在操作系统凭据库，不写入普通配置文件。
- 各平台生成提示词使用内置模板。
- 开发阶段提示词模板可见；产品完成后对用户隐藏。
- 提供整库备份与恢复，将热点、素材、创作项目、图片和配置打包为一个备份文件。
- 首版不做单项导入导出。
- 备份文件为单个压缩包，包含数据库、上传图片和普通配置，不包含 API Key。
- 备份不包含日志、临时文件和已有历史备份。
- 备份完成后允许用户另存到其他位置。
- 恢复前自动备份现有数据，再整体覆盖并重启应用。
- 首版不支持合并恢复。
- 备份包包含清单文件、工作区格式版本、数据库 Schema 版本和文件校验值。
- 同版本及旧版本备份允许恢复并执行迁移；来自更高工作区格式版本的备份拒绝恢复，不覆盖当前数据。
- 校验、迁移或重启失败时自动回滚到恢复前备份，并显示失败阶段和日志位置。
- 关于页提供手动检查更新；不后台自动下载、不强制升级。
- 检测到新版本后打开下载页面，由用户自行安装。
- 首次启动必须由用户选择业务数据目录。
- 数据库、图片、日志和应用内备份文件存放在所选目录，不存放在软件安装目录。
- 系统应用数据目录只保存启动所需的最小路径配置；API Key 仍由系统凭据机制保护。
- 首版设置页允许查看和打开数据目录，不允许直接修改；迁移通过备份和恢复完成。
- 启动时数据目录不可用或无写入权限，应用不得创建新的空数据库或进入主界面。
- 数据目录异常页提供重新选择目录、重试和退出。

### 4.7 日志

- 记录操作日志和错误日志。
- AI 请求记录模型、耗时、Token、请求状态和错误信息等元数据。
- 默认不长期保存完整提示词和模型原始响应。
- 提供默认关闭的调试模式；开启后可临时记录完整请求和响应，并支持一键清除。
- 日志默认保留 30 天，到期自动清理，也允许用户手动清空。

## 5. 搜索与筛选

- 首版不提供全局搜索。
- 热点、素材库和创作记录页面分别提供关键词搜索和基础筛选。

## 6. 导航结构

- 不设置“工作台”页面。
- 应用启动后默认进入“热点”页面。
- 已确认的一级导航：热点、素材库、创作、内容分发、统计、日志、设置。
- 各模块的内部页面结构仍在讨论中。

## 7. 界面与桌面行为

- 支持浅色、深色和跟随系统三种主题。
- 首版不使用系统托盘，关闭主窗口即退出应用。
- 存在热点刷新、AI 生成、备份或恢复任务时，关闭窗口前提示用户等待或强制退出。
- 应用只允许一个实例运行；重复启动时唤起已有窗口。
- 异常退出后恢复已经自动保存的编辑稿。
- 未完成的 AI 和热点任务不自动续跑，重新启动后标记为已中断，由用户手动重试。
- 自动保存失败时固定显示错误状态和重试按钮；恢复成功前，切换项目、返回列表和退出应用均需二次确认。
- 窗口最小尺寸为 900×640；Windows 使用 Ctrl、macOS 使用 Command 作为主快捷键修饰符。
- 首版快捷键：新建创作 `Ctrl/Cmd+N`、保存版本 `Ctrl/Cmd+S`、关闭抽屉或弹窗 `Esc`。
- 应用采用本地优先模式。
- 断网时已有热点浏览、素材管理、创作编辑、图片管理、分发复制、统计、日志和备份恢复保持可用。
- 断网或对应服务不可用时，只禁用热点刷新、链接正文读取和 AI 生成操作。

## 8. 暂不纳入首版

- 多用户、注册和登录。
- 自动发布到内容平台。
- AI 自动生成图片。
- 图片自动裁剪。
- 素材链接解析和文件上传。
- 全局搜索。
- 模型费用估算。
- 用户自定义提示词模板。
