# 决策记录

## 2026-07-15

| 决策 | 结果 |
|---|---|
| AI 联网搜索 | 不纳入首版；首版使用热点、素材库和手动输入补充资料 |
| 搜索筛选 | 模块内组合筛选，无结果时保留条件并提示清除方式 |
| 备份兼容 | 校验清单和版本；旧版本可迁移，高版本拒绝，失败自动回滚 |
| 保存失败 | 固定错误状态与重试入口，离开项目和退出应用前二次确认 |
| 窗口尺寸 | 最小 900×640 |
| 快捷键 | Windows 使用 Ctrl，macOS 使用 Command；首版支持新建、保存版本和 Esc 关闭 |
| M0 验收 | 第二轮交互原型与自动化回归通过后完成 M0 |

## 2026-07-11

| 决策 | 结果 |
|---|---|
| 产品形态 | 个人本地工作台，不做多用户 SaaS |
| 客户端 | Windows 和 macOS 跨平台桌面应用 |
| 首版平台 | 微信公众号、小红书、抖音、今日头条 |
| 抖音生成范围 | 不生成视频成片，只生成选题、作品描述、话题、标题、口播稿和分镜 |
| 平台选择 | 用户主动勾选需要生成的平台 |
| 热点方案 | 仅参考 TrendRadar 热点获取部分，可依赖公共热点 API |
| 热点正文 | 可选读取，默认关闭 |
| 素材录入 | 首版仅支持手动粘贴文本 |
| 图片能力 | 首版只生成提示词，用户分别上传各尺寸成品图 |
| AI 接口 | 仅支持 OpenAI 兼容接口 |
| 版本保留 | 当前版本和上一个版本 |
| 分发页职责 | 只展示、复制和记录，编辑统一在创作页完成 |

## 2026-07-12

| 决策 | 结果 |
|---|---|
| 创作方式 | 首次一次生成全部字段，各模块可独立刷新 |
| 创作选项 | 目标受众、风格、篇幅、立场、营销目的等作为可选项，并对所有所选平台统一生效 |
| 热点聚合 | 同一事件合并，保留各平台来源、排名和热度变化 |
| 热点保留 | 永久保存，允许手动清理；每 30 天提醒一次 |
| 热点刷新 | 仅手动刷新 |
| 创作入口 | 同时支持热点选题和自定义主题 |
| 素材引用 | 可多选，并区分重点参考与普通参考 |
| 提示词模板 | 内置；开发阶段可见，正式完成后隐藏 |
| 创作项目 | 一个主题对应一个项目，包含母稿、平台稿、图片和版本 |
| 母稿 | 在创作页可见、可编辑；修改后不自动更新平台稿 |
| Token 统计 | 仅按日期统计输入、输出和总 Token，不估算费用 |
| 日志 | 默认保存操作、错误和 AI 请求元数据；调试模式临时保存完整请求响应 |
| 备份恢复 | 首版支持整库备份与恢复 |
| 搜索 | 不做全局搜索，各主要模块独立搜索和筛选 |
| 删除方式 | 素材和创作项目二次确认后直接删除，无回收站 |
| 凭据保存 | API Key 使用系统凭据库加密保存 |
| 生成容错 | 平台任务相互独立，失败可单独重试 |
| 取消生成 | 只保留已经完整生成的平台结果 |
| 工作台页面 | 删除，应用默认进入热点页 |
| 热点页结构 | 分为聚合热点、平台榜单两个页签；统一时间筛选默认今天，历史数据通过筛选查询 |
| 热点详情 | 使用右侧抽屉，展示聚合摘要、趋势、来源明细、时间和正文读取开关，并提供素材与创作入口 |
| 热点刷新与聚合 | 先保存原始平台榜单再调用 AI；聚合失败不影响榜单查看，并提供重新聚合 |
| 素材库页面 | 使用列表和右侧新建/编辑抽屉；不提供用于创作入口，素材统一在创作设置中选择 |
| 素材字段 | 标题和正文必填，标签为可选自由文本且不复用；摘要截取正文，不调用 AI |
| 素材批量操作 | 支持批量删除，执行前必须二次确认 |
| 创作模块结构 | 使用创作项目列表和独立项目详情；详情分为创作设置、母稿、平台内容、图片素材四个页签 |
| 新建创作流程 | 按设置、母稿、平台内容、图片推进；页签可自由查看，但未满足前置条件时禁用生成操作 |
| 创作自动保存 | 停止输入约 1 秒后自动保存并显示状态；保存失败时离开页面需确认 |
| 创作版本 | 自动保存不产生版本；用户点击保存版本时，以整个项目建立新版本，始终只保留当前和上一版本 |
| 版本恢复 | 上一版本只读；可恢复为当前编辑稿，用户再次保存版本后才正式变更 |
| 平台内容布局 | 按已选平台分页签，每个字段独立编辑、复制、刷新和重试；只提供首次全部生成，不提供全部刷新 |
| 图片归属 | 图片绑定创作项目、平台和用途；每种封面规格一张，内容配图可多张并支持排序 |
| 内容配图定位 | 默认按顺序管理，可选绑定正文段落；分发页显示对应图片占位提示 |
| 分发状态 | 使用待分发、已复制、已发布；复制后自动标记，发布由用户手动标记，发布后修改需提示 |
| 分发列表 | 按创作项目显示父行，展开后显示平台任务子行；平台详情使用右侧抽屉 |
| 分发复制 | 只提供字段级复制；图片可单独复制或打开位置，不提供一键复制全部文本 |
| 统计页 | 展示四项总览及每日 Token 趋势，支持 7/30/90 天和自定义日期，并提供每日明细表 |
| 日志页 | 分为操作、错误、AI 调用三个页签；支持筛选，日志保留 30 天并允许手动清空 |
| 设置页 | 分为 AI 模型、数据管理、日志与调试、关于；提示词模板只在开发模式显示 |
| AI 配置数量 | 首版只保存一套通用 OpenAI 兼容配置和一个默认模型；MiMo 仅作为兼容验证目标 |
| 备份恢复 | 单压缩包备份且排除 API Key；恢复前自动备份，整体覆盖后重启，不支持合并 |
| 热点清理 | 支持清理 30 天前、90 天前、自定义日期前或全部；默认 30 天前，预览数量并二次确认 |
| 应用更新 | 首版只手动检查更新；发现新版本后打开下载页，不自动下载或强制升级 |
| 界面主题 | 首版支持浅色、深色和跟随系统 |
| 关闭行为 | 不使用系统托盘，关闭窗口即退出；运行中任务需要确认后才能强制退出 |
| 单实例 | 同一时间只运行一个应用实例，重复启动时唤起已有窗口 |
| 异常恢复 | 恢复自动保存编辑稿；未完成任务标记为已中断，不自动续跑，可手动重试 |
| 技术栈 | Electron + React + TypeScript + Vite + SQLite；UI 使用 Tailwind CSS 和 shadcn/ui，打包使用 Electron Forge |
| Electron 分层 | Renderer、Preload、Main 三层；本地能力通过白名单 IPC 调用，不启动本地 HTTP 服务或额外后台进程 |
| 数据访问 | better-sqlite3 + Drizzle ORM，SQLite 开启 WAL；主进程访问、启动迁移、关键写入使用事务 |
| 内容格式 | 微信公众号和今日头条使用 TipTap 富文本并保存 JSON/纯文本；小红书和抖音使用纯文本 |
| AI 任务队列 | 持久化队列，默认并发 2；区分六种状态，并按错误类型执行自动重试或直接失败 |
| AI 输出 | 首版不流式写入；完整返回并校验后显示，非结构化接口使用 JSON 解析并自动修复一次 |
| 上下文预算 | 重点素材优先，普通素材按需截取或 AI 压缩；提示用户并允许查看实际输入内容 |
| 热点接口 | 直接使用 NewsNow API；不集成 TrendRadar，仅参考其请求、校验和历史存储思路 |
| 热点源扩展 | 首版只实现 NewsNow 并预留统一适配器接口；多协议热点源及管理页面放到后续版本 |
| NewsNow 请求 | 唯一地址可配置；并发 3、超时 10 秒、重试 2 次，执行 Schema 与域名校验，允许部分成功 |
| 本地优先 | 可由确定性脚本完成的清洗、校验、转换和统计不调用 AI；AI 只处理语义和生成任务 |
| 热点聚合 | 本地清洗去重和候选分组，AI 判断事件归属并生成摘要；只做增量处理 |
| 图片资产 | 图片独立保存并由编辑稿和版本引用；仅在无任何引用时物理删除 |
| 项目删除 | 级联删除项目内容、版本、图片引用和分发记录；保留历史 Token 统计与日志，清理无引用图片 |
| 素材快照 | 项目保存所选素材的内容和优先级快照，素材库后续修改或删除不影响已有项目 |
| 热点快照 | 热点进入项目时保存聚合主题和来源明细快照，热点更新或清理不影响已有项目 |
| 分发数据源 | 分发页读取最新自动保存编辑稿；未保存版本时显示提示但允许复制 |
| 数据目录 | 首次启动由用户选择，业务数据全部存入所选目录；系统目录仅保存路径指针，首版通过备份恢复迁移 |
| 目录异常 | 数据目录不可用时阻止进入和建空库，提供重新选择、重试和退出 |
| 首次启动 | 数据目录必须选择；AI 配置可以跳过，未配置时禁用 AI 聚合和创作并提供引导 |
| 离线能力 | 采用本地优先模式，外部服务不可用只影响热点刷新、正文读取和 AI 生成 |
| 性能目标 | 支持百万热点、万级素材和项目；列表 300ms、搜索 500ms、冷启动 5 秒以内 |
| 中文搜索 | 首版使用 SQLite FTS5 trigram，1～2 字使用受限 LIKE；后续可增加中文分词器 |
| 代码组织 | 单应用仓库，分 main、preload、renderer、shared，并在进程内按业务 feature 组织 |
| 模型 Token 参数 | 高级设置可填上下文窗口和最大输出 Token；默认 32000/8000，用于预算和压缩 |
| AI 测试连接 | 发送最小生成请求并展示响应、耗时和 Token；测试用量计入统计 |
| 工作区结构 | 用户目录下建立 SelfMediaWorkbench，包含标识、数据库、图片、日志、备份和临时目录 |
| 备份内容 | 包含工作区标识、数据库、图片和普通配置；排除凭据、日志、临时文件和历史备份 |
| 正文读取 | 使用 HTTP + Readability 通用提取，不做浏览器自动化；失败可手动粘贴并回退已有热点信息 |
| 正文读取时机 | 仅在创作阶段由用户勾选后执行，结果保存到项目快照；热点浏览不读取，首版无全局缓存 |
| 正文来源选择 | 用户手动多选，默认不选；提供选择排名最高三个来源的快捷操作 |
| 平台规格 | 使用内置 PlatformSpec 统一字段、限制、图片规格、编辑器和复制顺序，并随应用升级 |
| 超限处理 | 自动 AI 压缩一次，仍超限由脚本按字段类型裁剪；最终结果必须满足平台限制 |
| 配图提示词 | 根据正文自动生成并绑定段落，每个平台最多 8 个；用户可增删或单独刷新 |
| 项目状态 | 系统自动计算草稿、创作中、待分发、分发中、已完成；生成失败单独标记 |
| 复制项目 | 复制当前编辑稿、参考快照、平台内容和图片引用；不复制版本、分发记录及发布状态 |
| 新建创作入口 | 左侧导航顶部固定按钮；空白和热点入口复用同一流程，素材只在创作设置中选择 |
| 平台榜单 | 按平台页签分别展示，不混排不同平台排名 |
| 创作参考 | 热点可不选或多选，也可启用 AI 联网搜索；素材在创作设置中勾选并仅用于母稿上下文 |
| 参考选择交互 | 热点和素材使用可搜索弹窗，创作页面只展示已选摘要 |
