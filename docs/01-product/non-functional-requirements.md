# 非功能需求

> 状态：草案  
> 最近更新：2026-07-13

## 性能

| 指标 | 首版目标 |
|---|---|
| 原始热点记录 | 100 万条以内保持正常查询 |
| 创作项目 | 1 万个以内 |
| 素材 | 1 万条以内 |
| 普通列表查询 | 300ms 内 |
| 搜索与筛选 | 500ms 内 |
| 应用冷启动 | 5 秒内进入可操作界面 |
| 自动保存 | 停止输入后约 1 秒完成 |

网络请求和 AI 模型生成耗时不计入本地响应目标。

### 搜索

- 中文关键词搜索使用 SQLite FTS5 trigram 索引。
- 1～2 字短查询使用受限 LIKE 兜底。
- 首版不引入独立搜索服务。

## 可用性

- 应用采用本地优先模式，断网不影响本地数据浏览和编辑。
- 数据目录异常时不得静默创建空数据库。
- 外部服务部分失败不得影响已经成功的数据或其他本地模块。
- 异常退出后恢复自动保存编辑稿，运行中任务标记为已中断。

## 数据可靠性

- SQLite 使用 WAL、外键约束和事务。
- 项目保存版本、删除、备份和恢复必须保持原子性或具备明确回滚结果。
- 图片使用资产引用管理，避免版本恢复时文件丢失。
- 提供整库备份、恢复前自动备份和备份完整性校验。

## 安全与隐私

- API Key 不进入普通配置、数据库、备份和日志。
- Renderer 不直接访问 Node.js、数据库和文件系统。
- 外部链接正文读取限制协议、域名、大小、超时和重定向。
- 调试日志默认关闭，并允许一键清理。

## 兼容性

- 支持 Windows 和 macOS。
- 支持浅色、深色和跟随系统主题。
- 安装、更新和备份恢复需要覆盖两个操作系统的测试。
- 窗口最小尺寸为 900×640。
- 主快捷键在 Windows 使用 Ctrl，在 macOS 使用 Command。
