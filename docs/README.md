## 维护规则

- 已由用户确认的内容写入《产品需求》和《决策记录》。
- 尚未确认的设计只写入《待确认事项》，不得作为开发依据。
- 需求发生变化时直接更新《产品需求》，并在《决策记录》中追加变更，不覆盖历史决策。
- 日期以用户所在时区 Asia/Hong_Kong 为准。
- 文档状态统一使用：待编写、草案、讨论中、已确认、已废弃。
