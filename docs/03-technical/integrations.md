# 外部接口

## 公共热点 API

- 直接使用 NewsNow API 获取平台热榜。
- 不引入 TrendRadar 代码或运行时依赖；只参考其 NewsNow 调用和数据处理思路。
- 请求形式按 NewsNow source ID 获取各平台最新榜单。
- 需要设计请求频率限制、超时、重试、数据校验和 API 不可用时的提示。

### 热点源扩展边界

- 首版只实现 NewsNowAdapter。
- 内部预留统一适配器接口：测试连接、获取可用平台、获取榜单、转换统一数据格式。
- 首版不提供热点源管理页面，也不实现其他协议适配器。
- 后续版本可加入多数据源实例管理及跨源标准化、去重。

### NewsNow 首版配置

- 数据管理高级设置提供唯一的 NewsNow API 地址，默认使用公共实例。
- 平台请求最大并发数为 3。
- 单次请求超时 10 秒，失败最多重试 2 次。
- 使用 Zod 校验响应 Schema。
- 原始内容链接必须使用 HTTPS，并匹配平台域名白名单。
- 单个平台失败不影响其他平台数据保存。

## 热点链接正文读取

- 只在用户进入创作并主动勾选后执行，热点列表和详情浏览不读取正文。
- 主进程通过 HTTP 获取 HTML，并限制协议、域名、重定向、响应大小和超时。
- 使用 Mozilla Readability 提取标题和正文，再由本地规则清洗广告、导航和空白。
- 不启动浏览器自动化，不处理登录、验证码或强反爬页面。
- 提取失败时使用热点标题和已有摘要，并允许用户手动粘贴正文。
- 成功结果进入项目热点快照。
- 首版不建立跨项目的全局正文缓存。
- 聚合热点的正文读取按用户选择的来源执行，并提供选择排名最高三个来源的快捷规则。

## OpenAI 兼容接口

- 配置项：Base URL、API Key、模型名。
- 提供测试连接。
- 首版只保存一套配置和一个默认模型。
- 保持供应商无关，不内置 MiMo 专用协议；开发与测试阶段优先验证小米 MiMo。
- 需要兼容不同服务商在响应格式和 Token 用量字段上的差异。
- Base URL 默认视为 API 根路径，测试连接在其后追加 `/chat/completions`；也允许直接填写完整 Chat Completions 地址。

## 系统能力

- Windows Credential Manager。
- macOS Keychain。
- 本地文件选择、备份导出和恢复导入。
